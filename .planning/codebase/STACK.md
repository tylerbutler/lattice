# Technology Stack

## Languages

**Primary:**
- Gleam 1.14.0 - Core library implementation targeting Erlang (BEAM) runtime

**Secondary:**
- Erlang 27.2.1 - BEAM VM runtime environment

## Runtime

**Environment:**
- Erlang/OTP 27.2.1 (BEAM VM)
- Target platform: Erlang VM (distributed, fault-tolerant runtime)

**Package Manager:**
- Gleam's built-in dependency management (hex.pm)
- Lockfile: `manifest.toml` (auto-generated)

## Frameworks

**Core:**
- Gleam stdlib 0.68.1 - Standard library (functional programming utilities)

**Testing:**
- gleeunit 1.9.0 - Gleam test framework with assertion library

**Build/Dev:**
- just 1.38.0 - Command runner for standardized development tasks
- changie 1.0+ - Changelog management tool

## Key Dependencies

**Critical:**
- gleam_stdlib >=0.48.0 <2.0.0 - Runtime library with core functions (Result, List, String, etc.)
- gleeunit >=1.0.0 <2.0.0 - Test runner and assertion library (`should` module)

## Configuration

**Environment:**
- `.tool-versions` - Tool version lock file (source of truth for CI)
  - Erlang 27.2.1
  - Gleam 1.14.0
  - just 1.38.0
- `.mise.toml` - Mise configuration (alternative to `.tool-versions`)
  - Tools: changie, erlang, gleam, just
  - Locale: en_US.UTF-8

**Build:**
- `gleam.toml` - Project manifest with metadata, dependencies, version
- `manifest.toml` - Locked dependency versions (generated by `gleam deps download`)
- `justfile` - Task definitions for development workflow

**Code Style:**
- `.editorconfig` - Unified editor configuration
  - 2-space indentation (Gleam, TOML, JSON, YAML)
  - UTF-8 encoding, LF line endings
  - Trailing whitespace trimmed
- Built-in `gleam format` (automatic code formatter)
- No external linting tool (Gleam enforces via compiler)

## Platform Requirements

**Development:**
- Erlang 27+ (OTP runtime)
- Gleam 1.7+ (compiler and toolchain)
- just 1.38+ (task runner)
- mise or asdf (optional, for tool version management)

**Production:**
- Erlang/OTP 27.2.1
- BEAM VM for runtime execution
- Deployment to Hex.pm (Gleam package registry)

## Build Artifacts

- `build/` directory (Git-ignored) - Compiled artifacts and intermediate files
- Generated on `gleam build` and `gleam test`
- Cleaned via `just clean`

## Version Management

**Release:**
- Semantic versioning (MAJOR.MINOR.PATCH)
- Source of truth: `version` field in `gleam.toml`
- Automated via release-please + changie
- Changelog managed in `.changes/` directory (unreleased change fragments)
- Merged changelog written to `CHANGELOG.md`

## Package Distribution

**Registry:**
- Hex.pm (official Gleam package repository)
- Published via `gleam publish` with HEXPM_API_KEY secret
- Automatically triggered on GitHub release (tag v*)
