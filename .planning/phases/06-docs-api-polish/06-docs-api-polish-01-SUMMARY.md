## Decisions Made
- **GCounter/PNCounter non-opaque:** pn_counter directly destructures GCounter in to_json/from_json (lines constructing `g_counter.GCounter(dict:, self_id:)`) — making opaque would require adding public accessors for raw dict and self_id, which would be worse API design than the current state.
- **LWWRegister non-opaque:** value and timestamp fields are part of the public API contract; users legitimately access `register.value` and `register.timestamp` directly.
- **MVRegister opaque:** Users only need `new`, `set`, `value`, and `merge`; internal entries dict and vclock are pure implementation details.
- **Tag opaque:** Tags are generated internally by `set()`; no test or external module constructs a `Tag` directly.
- **VersionVector opaque + to_dict/from_dict:** Added two public escape hatches to support mv_register serialization without leaking the internal dict field. This is a cleaner pattern than leaving the constructor public.
- **DotContext opaque:** All external uses go through API functions; Dot remains public since users pass Dot values to contains_dots/remove_dots.
